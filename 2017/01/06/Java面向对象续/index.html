Java面向对象续

发表于 2017-01-06 | 更新于 2017-01-06 | 分类于 Java | | 阅读次数

基本数据类型:

  • java为8个基本的数据类型提供了对应的包装类，可以使其对应变成对象使用。自动装箱和自动拆箱，允许把基本类型值直接赋给对应的包装类引用变量，也允许把包装类对象直接赋给对应的基本类型变量。基本数据类型变量->包装类实例，是通过对应包装类的构造器来实现的。除了Character之外，还可以通过传入一个字符串参数来构建包装类对象。

    | byte | short | int | long | char | float | double | boolean |
    | —- | —– | ——- | —- | ——— | —– | —— | ——- |
    | Byte | Short | Integer | Long | Character | Float | Double | Boolean |

  • 包装类可实现基本类型变量和字符串之间的转换，除了Character之外的所有包装类，都提供了一个parseXxx(String s)静态变量， 用于将一个特定字符串转换成基本类型变量,在String 类里也提供了多个重载valueOf()方法，用于将基本类型变量转换成字符串。

  • toString()方法：输出对象的“自我描述”信息，用以告诉外界该对象具有的状态信息。Object类提供的toString()方法总是返回该对象实现类的：类名＋@＋hashCode值，这个返回值不能够真实的实现“自我描述”的功能，常重写该方法，来实现对象的“自我描述”。

  • ==和equals:==如果两个变量是基本类型变量，且都是数值型，则需要两个变量的值相等，等号判断就将返回true；两个引用类型的变量，必须指向同一个对象时，==等号判断才会返回true.例如：65和65.0，’A’相等。String重写的equals():只要两个字符串所包含的字符序列相同，通过equals()比较将返回true；否则将返回false。但是Object类也同样提供一个实例方法equals()，因此所有引用变量都可以调用该方法来判断是否与其它引用变量相等。此时的equals()方法与等号没有区别，同样要求两个引用变量指向同一个对象才会返回true.

  • java利用final关键字来修饰变量、方法和类，系统不允许为final变量重新赋值，子类不允许覆盖父类的final方法，final不允许派生子类。即利用final实现java的不可变类。

  • abstract 和interface 两个关键字分别用于定义抽象类和接口，抽象类和接口都是从多个类中抽象出来的共同特征。抽象类主要作为多个类的模版，接口则定义了多类应用遵守的规范。

  • enum关键字则用于创建枚举类型，枚举类是一种不能自由创建对象的类，枚举类是一种不能自由创建对象的类，枚举里的对象在定义类时已经固定下来。

    类成员:

    static：修饰的成员就是类成员,类成员属于整个类，而不属于单个对象。类的属性既是可通过类来访问，也可通过类的对象来访问(==但通过对象访问的依然是该类的属性==)。

    单实例类：如果一个类始终只能创建一个实例，则这个类被称为单实例类。
