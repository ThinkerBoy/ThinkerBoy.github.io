2016年度总结

发表于 2017-01-05 | 更新于 2017-01-13 | 分类于 感悟 | | 阅读次数

​ 2015年3月27日，参与复试答辩，4月8日来到计算所，开始学习，我清楚的记得刚进所的时候，分配的任务是配合师兄，做关于船舶轨迹挖掘的相关研究以及数据可视化，其实给我的任务只有前者，但是我将可视化的这个任务也拦过来了。由于刚接触关于船舶的方面的东西，不是太过了解其数据属性和业务背景等等，就开始自己要了些资料以及看了些论文补充自己在这方面的欠缺。当时师兄正字做一个基于网格计算的船舶轨迹的挖掘，他负责整个框架的设计，而我是接手之前的一个同学写了一半的算法，在完善了整个程序框架和算法的对接后，我开始了数据集的测试，用了2014年的10、11、12月三个月的船舶数据做测试，其单月数据在4亿多条。由于一开始不是太了解之前同学写的算法的具体内涵，跑一个月的4亿条数据时，网格划分是1度(64800个网格)的情况下，跑了25个小时左右，当时想在服务器上多线程跑，却这么慢，跑完三个月的数据用了近一周左右。然后我就用Matlab写了一个heatmap算法，来可视化之前的数据，从而观察基于网格计算船舶轨迹的算法是否可以看到一些有用的现象。当要进行0.1、0.5、0.1、0.01度的网格计算时，程序跑了几天都没有跑完，因为当达到0.01度时就是4亿条数据，1.64亿个网格进行特征参数的计算。于是就想了，更改之前同学的算法，在翻阅了几篇网格计算的相关论文和仔细研究之前同学写的算法之后，发现之前同学用的是基于插值的算法，也就是利用微积分的思想，将相邻的两个船舶点划分成了无数点，然后来计算的特征参数。
